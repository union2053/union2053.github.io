C Functions with a Variable Number of Arguments

PreviousNext

Some functions in the standard libraries accept a variable number of arguments.

The functions printf() and scanf_s() are obvious examples.

The standard library stdarg.h provides you with routines to write some of your own.

Suppose you're going to produce a function to calculate the average of two or more values of type double.

The prototype would be written as follows: 

double average(double v1, double v2, ...);

The ellipsis (...) indicates that a variable number of arguments may follow the first two fixed arguments.

You must have at least one fixed argument.

The remaining specifications are as you would usually find with a function prototype.

In the example above, the first two arguments are of type double, and the function returns a double result.

To reference the extra arguments we use pointers.

The stdarg.h library header provides those pointers.

functions

You need to use the following to implement your own function with a variable number of arguments.

  • va_start()
  • va_arg()
  • va_end().

The first of these has the following form: 

void va_start(va_list parg, last_fixed_arg);

The va_start function accepts two arguments:

  • a pointer parg of type va_list and
  • the name of the last fixed parameter you specified for the function you're writing.

The va_list type is a type defined in stdarg.h and is designed to store information required by the routines that support variable argument lists.

So using the function average() as an illustration, you can start to write the function as follows: 

double average(double v1, double v2,...) 
{ 
  va_list parg;              // Pointer for variable argument list 
  // More code to go here... 
  va_start( parg, v2); 
  // More code to go here... 
}

You first declare the variable parg of type va_list.

You then call va_start() with this as the first argument and specify the last fixed parameter v2 as the second argument.

Calling va_start() sets the variable parg to point to the first variable argument that is passed to the function when it is called.

You don't know what type of value this represents, and the standard library does not provide any ways to get the type.

You must determine the type of each variable argument.

The average() function deals with arguments of type double.

You now need to know how to access the value of each of the variable arguments as follows: 

// Function to calculate the average of two or more arguments 
double average( double v1, double v2,...) 
{ 
  va_list parg;              // Pointer for variable argument list 
  double sum = v1 + v2;      // Accumulate sum of the arguments 
  double value = 0.0;        // Argument value 
  int count = 2;             // Count of number of arguments 

  va_start(parg,v2);         // Initialize argument pointer 
  while((value = va_arg(parg, double)) != 0.0) 
  { 
    sum += value; 
    ++count; 
  } 
  va_end(parg);              // End variable argument process 
  return sum/count; 
}

After declaring parg, you declare the variable sum as double.

The next variable, value, declared as double will be used to store the values of the variable arguments as you obtain them one by one.

You then declare a counter, count, for the total number of arguments and initialize this with the value 2 because you know you have at least that many values from the fixed arguments.

After you call va_start() to initialize parg, most of the action takes place within the while loop.

The while loop condition calls another function from stdarg.h, va_arg().

The first argument to va_arg() is the variable parg you initialized through the call to va_start().

The second argument is the type of the argument you expect to find.

The function va_arg() returns the value of the current argument specified by parg, and this is stored in value.

It also updates the pointer parg to point to the next argument in the list, based on the type you specified in the call.

If you don't specify the correct type, you won't be able to obtain the next argument correctly.

You assume that all the arguments will be nonzero except for the last.

This is reflected in the condition for continuing the loop, being that value isn't equal to 0.

When the loop ends, and you execute the statement 

va_end(parg);                // End variable argument process

va_end() ends the process. It resets the parg pointer to NULL.

If you omit this call, your program may not work properly.

Using Variable Argument Lists

Calculating an average using variable argument lists 

#include <stdio.h> 
#include <stdarg.h> 

double average(double v1 , double v2,...);       // Function prototype 

int main(void) 
{ 
  double v1 = 10.5, v2 = 2.5; 
  int num1 = 6, num2 = 5; 
  long num3 = 12L, num4 = 20L; 

  printf("Average = %.2lf\n", average(v1, 1233.5, v2, 3214.5, 0.0)); 
  printf("Average = %.2lf\n", average(1123.0, 2321.0, 0.0)); 
  printf("Average = %.2lf\n", average( (double)num2, v2,(double)num1, 
                                      (double)num4,(double)num3, 0.0)); 
  return 0; 
} 

// Function to calculate the average of two or more arguments 
double average( double v1, double v2,...) 
{ 
  va_list parg;              // Pointer for variable argument list 
  double sum = v1 + v2;      // Accumulate sum of the arguments 
  double value = 0.0;        // Argument value 
  int count = 2;             // Count of number of arguments 

  va_start(parg,v2);         // Initialize argument pointer 
  while((value = va_arg(parg, double)) != 0.0) 
  { 
    sum += value; 
    ++count; 
  } 
  va_end(parg);              // End variable argument process 
  return sum/count; 
}
