C Manifest Constants: #define

PreviousNext

The #define preprocessor directive begins with the # symbol at the beginning of a line.

The ANSI and subsequent standards permit the # symbol to be preceded by spaces or tabs, and it allows for space between the # and the remainder of the directive.

A directive can appear anywhere in the source file, and the definition holds from its place of appearance to the end of the file.

You can spread the directive over several physical lines.

These lines, however, constitute a single logical line.

Syntax

Each #define line has three parts.

The first part is the #define directive itself.

The second part is your chosen abbreviation, known as a macro.

Some macros represent values, we call them object macro.

The macro name must have no spaces in it, and it must conform to the same naming rules that C variables follow: Only letters, digits, and the underscore (_) character can be used, and the first character cannot be a digit.

The third part, the remainder of the line, is termed the replacement list or body. 


#include <stdio.h>
#define TWO 2        /* you can use comments if you like   */
/* a backslash continues a definition 
to the next  line   */
#define OW "This is a long\
long long line"

#define FOUR TWO*TWO
#define PX printf("X is %d.",123);

int main(void)
{
    int x = TWO;

    PX;
    x = FOUR;
    printf("%s\n", OW);
    printf("TWO: OW\n");

    return 0;
}

Output:

When the preprocessor finds an example of one of your macros within your program, it replaces it with the body.

This process of going from a macro to a final replacement is called macro expansion.

Note that you can use standard C comments on a #define line; each is replaced by a space before the preprocessor sees it.
