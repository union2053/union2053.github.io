C Concept of a File

PreviousNext

A file is essentially a serial sequence of bytes.


Positions in a File

A file has a beginning and an end, and it has a current position, typically defined as so many bytes from the beginning.

The current position is where any file action, read or write, will take place.

You can move the current position to any point in the file.

A new current position can be specified as an offset from the beginning of the file or.

You can also move the position to the end of the file.

File Streams

The C library provides functions for reading and writing to or from data streams.

A stream is an abstract representation of any external source or destination for data.

The keyboard, the command line on your display, and files are all examples of things you can work with as streams.

You use the same input/output functions for reading and writing any external device that is mapped to a stream.

Text vs Binary

There are two ways of writing data to a stream that represents a file.

First, you can write a file as a text file, in which case data are written as a sequence of characters organized as lines, where each line is terminated by a newline character.

Second, you can write a file as a binary file. Data written to a binary file are written as a series of bytes exactly as they appear in memory, so a value of type double, for example, would be written as the 8 bytes that appear in memory.

Accessing Files

The files that are on your disk drive each have a name, and the rules for naming files are determined by your operating system.

File pointer

A file pointer points to a struct of type FILE that represents a stream.

The FILE structure contains information about the file.

If you want to use several files simultaneously in a program, you need a separate file pointer for each file.

As soon as you've finished using one file, you can associate the file pointer you were using with another file.

If you need to process several files, but you'll be working with them one at a time, you can do it with one file pointer.
