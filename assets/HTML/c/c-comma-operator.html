C Comma Operator

PreviousNext

The comma operator extends the flexibility of the for loop by enabling you to include more than one initialization or update expression in a single for loop specification.

For example, the following program prints first-class postage rates. 

#include <stdio.h>
int main(void)
{
    const int FIRST_OZ = 46; // 2013 rate
    const int NEXT_OZ = 20; // 2013 rate
    int ounces, cost;

    printf(" ounces cost\n");
    for (ounces=1, cost=FIRST_OZ; ounces <= 16; ounces++,
         cost += NEXT_OZ)
        printf("%5d   $%4.2f\n", ounces, cost/100.0);

    return 0;
}

The program uses the comma operator in the initialize and the update expressions.

Its presence in the first expression causes ounces and cost to be initialized.

Its second occurrence causes ounces to be increased by 1 and cost to be increased by 20 (the value of NEXT_OZ) for each iteration.

All the calculations are done in the for loop specifications.

The comma operator is not restricted to for loops, but that's where it is most often used.

Properties

The operator has two further properties.

First, it guarantees that the expressions it separates are evaluated in a left-to-right order.

Suppose, for instance, that you had this expression: 

ounces++, cost = ounces * FIRST_OZ

This would increment ounces and then use the new value for ounces in the second subexpression.

The comma being a sequence point guarantees that the side effects of the left subexpression occur before the right subexpression is evaluated.

Second, the value of the whole comma expression is the value of the right-hand member.

The effect of the statement 

x = (y = 3, (z = ++y + 2) + 5);

is to first assign 3 to y, increment y to 4, and then add 2 to 4 and assign the resulting value of 6 to z, next add 5 to z, and finally assign the resulting value of 11 to x.

Suppose you use comma notation in writing a number: 

price = 249,500;

This is not a syntax error. Instead, C interprets this as a comma expression, with price = 249 being the left subexpression and 500 the right subexpression.

Therefore, the value of the whole comma expression is the value of the right-hand expression, and the left sub statement assigns the value 249 to the price variable.

Therefore, the effect is the same as the following code: 

price = 249;
500;

Remember that any expression becomes a statement with the addition of a semicolon, so 500; is a statement that does nothing.

On the other hand, the statement 

price = (249,500);

assigns 500, the value of the right subexpression, to price.

The comma also is used as a separator, so the commas in 

char ch1, date1;

and 

printf("%d %d\n", int1, int2);

are separators, not comma operators.
