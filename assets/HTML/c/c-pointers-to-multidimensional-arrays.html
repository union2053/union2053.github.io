C Pointers to Multidimensional Arrays

PreviousNext

To declare a pointer variable pz that can point to a two-dimensional array such as my_array[][].

pz must point to an array of two ints, not to a single int.

Here is what you can do: 

int (* pz)[2];     // pz points to an array of 2 ints

This statement says that pz is a pointer to an array of two ints.

[] has a higher precedence than *.

Therefore, with a declaration such as 

int * px[2];     // px is an array of two pointers-to-int

you apply the brackets first, making px an array of two something.

Next, you apply the *, making px an array of two pointers.

Finally, use the int, making px an array of two pointers to int.

This declaration creates two pointers to single ints, but the original version uses parentheses to apply the * first, creating one pointer to an array of two ints.

The following code shows how you can use such a pointer just like the original array. 

#include <stdio.h>
int main(void)
{
    int my_array[4][2] = { {2,1}, {6,3}, {1,9}, {5, 7} };
    int (*pz)[2];
    pz = my_array;

    printf("      pz = %p,      pz + 1 = %p\n",
          pz,          pz + 1);

    printf("pz[0] = %p, pz[0] + 1 = %p\n",
             pz[0],       pz[0] + 1);
    printf(" *pz = %p,      *pz + 1 = %p\n",
               *pz,         *pz + 1);
    printf("pz[0][0] = %d\n", pz[0][0]);
    printf(" *pz[0] = %d\n", *pz[0]);
    printf("     **pz = %d\n", **pz);
    printf("        pz[2][1] = %d\n", pz[2][1]);
    printf("*(*(pz+2) + 1) = %d\n", *(*(pz+2) + 1));

    return 0;
}

Output:

You can use notation such as pz[2][1], even though pz is a pointer, not an array name.

More generally, you can represent individual elements by using array notation or pointer notation with either an array name or a pointer: 

my_array[m][n] == *(*(my_array + m) + n)
pz[m][n] == *(*(pz + m) + n)
