C float, double, and long double

PreviousNext

C type float, double, or long double correspond to the real types.

The floating-point approach enables you to represent a much greater range of numbers, including decimal fractions.

Floating-point number representation is similar to scientific notation , a system used by scientists to express very large and very small numbers.

In scientific notation, numbers are represented as decimal numbers times powers of 10.

Here are some examples.

Number Exponential Notation
1,000,000,000 1.0e9
123,000 1.23e5
322.56 3.2256e2
0.000056 5.6e-5

The first column shows the usual notation, the second column exponential notation, or e-notation, which is the way scientific notation is usually written for and by computers, with the e followed by the power of 10.

double

C has a double floating-point type. The double type has the same minimum range requirements as float , but it extends the minimum number of significant figures.

Typical double representations use 64 bits instead of 32.

C has a third floating-point type: long double.

It provides for even more precision than double.

However, C guarantees only that long double is at least as precise as double.

The three types of floating-point variables are listed in the following table.

Keyword
 Number of bytes
 Range of values
float
 4
 +/-3.4E+/-38 (6-7 decimal digits of precision)
double
 8
 +/-1.7E+/-308 (15 decimal digits of precision)
long double
 12

Declaring Floating-Point Variables

Floating-point variables are declared and initialized in the same manner as integer.

Here are some examples: 

float          n, j;
double         t;
float          p = 6.63e-34;
long double    g;

Floating-Point Constants (Literals)

To write a constant of type float, you append an f to the number to distinguish it from type double.

You could initialize the variables when you declare them like this: 

float radius = 2.5f; 
double biggest = 123E30;

To specify a long double constant, you append an uppercase or lowercase letter L, as in the following example: 

long double huge = 1234567.89123L;

The basic form of a floating-point literal is a signed series of digits, including a decimal point, followed by an e or E, followed by a signed exponent indicating the power of 10 used.

Here are two valid floating-point constants: 

-1.56E+12
2.87e-3

You can leave out positive signs. You can do without a decimal point (2E5) or an exponential part (123.456), but not both simultaneously.

You can omit a fractional part (3.E16) or an integer part (.45E-6), but not both.

Here are some more valid floating-point constants: 

3.14159
.2
4e16
.8E-5
100.

Don't use spaces in a floating-point constant. 

Wrong: 1.56 E+12

By default, the compiler assumes floating-point constants are double precision.

Suppose, for example, that some is a float variable and that you have the following statement: 

double some = 4.0 * 2.0;

Then 4.0 and 2.0 are stored as double , using typically 64 bits for each.

The product is calculated using double precision arithmetic, and only then is the answer trimmed to regular float size.

This ensures greater precision for your calculations.

C enables you to override this default by using an f or F suffix to make the compiler treat a floating-point constant as type float; examples are 2.3f and 9.11E9F.

If the floating-point number has no suffix, it is type double.

Since C99, C has a new format for expressing floating-point constants.

It uses a hexadecimal prefix ( 0x or 0X ) with hexadecimal digits, a p or P instead of e or E, and an exponent that is a power of 2 instead of a power of 10.

Here's what such a number might look like: 

0xa.1fp10
