Integer output variations 

#include <stdio.h> 

int main(void) 
{ 
  int        i = 15,       j = 345,            k = 4567; 
  long long li = 56789LL, lj = 98765432167LL, lk = 23456789LL; 

  printf("i = %d   j = %d    k = %d   i = %6.3d   j = %6.3d   k = %6.3d\n", 
                                                     i ,j, k, i, j, k); 
  printf("i = %-d   j = %+d   k = %-d   i = %-6.3d   j = %-6.3d   k =" 
                                         " %-6.3d\n", i ,j, k, i, j, k); 
  printf("li = %d   lj = %d   lk = %d\n", li, lj, lk); 
  printf("li = %lld   lj = %lld   lk = %lld\n", li, lj, lk); 
  return 0; 
}

Output:

You may get warnings from your compiler relating to the third output statement.

The - flag causes the output to be left justified.

You get a leading plus sign in the output of j on the second line because of the flag modifier.

You can use more than one flag modifier if you want.

The problem arises only if long long and int are of different sizes.

You get the correct values from this statement: 

printf("li = %lld   lj = %lld   lk = %lld\n", li, lj, lk);
