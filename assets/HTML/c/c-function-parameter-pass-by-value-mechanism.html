C Function Parameter Pass-by-Value Mechanism

PreviousNext

When you pass an argument to a function, the argument itself, is not passed directly to the function.

A copy of the argument value is made first and stored on the stack, and this copy is made available to the function, not the original value.

Note: the stack is a part of the memory that each function has to persist local variables and arguments from the signature.

Example

The following code is trying to change the value of the parameter in function. 

#include <stdio.h> 

void my(int n) 
{ 
  printf("Inside function: n is now %d \n",n); 
  n = 5;
  printf("Inside function: n is now %d \n",n); 
} 
// main program - execution always starts here 
int main(void) 
{ 
  int n = 0; 
  printf("main: n is now %d \n",n); 
  my(n); 
  printf("main: n is now %d \n",n); 

  return 0; 
}

Output:

From the output we can see that the value of the parameter is changed in my() but the value will not stay.

Outside the my() function, n is still the zero, which is not changed by the my().
