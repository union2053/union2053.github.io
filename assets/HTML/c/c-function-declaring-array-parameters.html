C function Declaring Array Parameters

PreviousNext

Because the name of an array is the address of the first element, an actual argument of an array name requires that the matching formal argument be a pointer.

In this context, and only in this context, C interprets int ar[] to mean the same as int *ar; that is, ar is type pointer-to-int.

Because prototypes allow you to omit a name, all four of the following prototypes are equivalent: 

int sum(int   *ar, int n);
int sum(int   *, int);
int sum(int   ar[], int n);
int sum(int   [], int);

You can't omit names in function definitions, so, for definitions, the following two forms are equivalent: 

int sum(int *ar, int n)
{
    // code goes here
}

int sum(int ar[], int n);
{
    // code goes here
}

You should be able to use any of the four prototypes with either of the two definitions shown here.

Example

The following code shows a program using the sum() function.

To point out an interesting fact about array arguments, the program also prints the size of the original array and the size of the function parameter representing the array. 

// use %u or %lu if %zd doesn't work
#include <stdio.h>
#define SIZE 10
int sum(int ar[], int n);
int main(void)
{
    int my_array[SIZE] = {21,12,52,39,44,16,19,26,31,20};
    long answer;

    answer = sum(my_array, SIZE);
    printf("The total number of my_array is %ld.\n", answer);
    printf("The size of my_array is %zd bytes.\n",
          sizeof my_array);

    return 0;
}

int sum(int ar[], int n)          // how big an array?
{
    int i;
    int total = 0;

    for( i = 0; i < n; i++)
        total += ar[i];
    printf("The size of ar is %zd bytes.\n", sizeof ar);

    return total;
}

Output:

Note

The size of my_array is 40 bytes.

This makes sense because my_array contains 10 ints, each 4 bytes, for a total of 40 bytes.

But the size of ar is just 8 bytes.

That's because ar is not an array itself; it is a pointer to the first element of my_array.

Our system uses 8 bytes for storing addresses, so the size of a pointer variable is 8 bytes.

In short, my_array is an array, ar is a pointer to the first element of my_array, and the C connection between arrays and pointers lets you use array notation with the pointer ar.
