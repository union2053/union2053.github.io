C Creating Strings from Macro Arguments: The # Operator

PreviousNext

Here's a function-like macro: 

#define PSQR(X)      printf("The square of X is %d.\n", ((X)*(X)));

Suppose you used the macro like this: 

PSQR(8);

Here's the output: 

The square of X is 64.

The X in the quoted string is treated as ordinary text, not as a token that can be replaced.

To include the macro argument in a string, use the # symbol.

It converts tokens into strings.

For example, say that x is a macro parameter, and then #x is that parameter name converted to the string "x".

This process is called stringizing.

Example 

#include <stdio.h>
#define PSQR(x) printf("The square of " #x " is %d.\n",((x)*(x)))

int main(void)
{
    int y = 5;

    PSQR(y);
    PSQR(2 + 4);

    return 0;
}

Here's the output:

In the first call to the macro, #x was replaced by "y", and in the second call #x was replaced by "2 + 4".

ANSI C string concatenation then combined these strings with the other strings in the printf() statement to produce the final strings that were used.

For example, the first invocation becomes this: 

printf("The square of " "y" " is %d.\n",((y)*(y)));

Then string concatenation converts the three adjacent strings to one string: 

"The square of y is %d.\n"
