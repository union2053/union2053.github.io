C Character String Arrays and Initialization

PreviousNext

When you define a character string array, you must let the compiler know how much space is needed.

One way is to specify an array size large enough to hold the string.

The following declaration initializes the array m1 to the characters of the indicated string: 

const char m1[40] = "this is a test.";

The const indicates the intent to not alter this string.

This form of initialization is short for the standard array initialization form.

Without the closing null character, you have a character array, but not a string.

When you specify the array size, be sure that the number of elements is at least one more (null ending character) than the string length.

Any unused elements are automatically initialized to 0, which in char form is the null character, not the zero digit character.

It is convenient to let the compiler determine the array size.

If you omit the size in an initializing declaration, the compiler determines the size for you: 

const char m2[] = "this is a test from demo2s.com.";

Letting the compiler compute the size of the array works only if you initialize the array.

If you create an array you intend to fill later, you need to specify the size when you declare it.

When you do declare an array size, the array size must evaluate to an integer.

Prior to the advent of variable length arrays (VLAs) with C99, the size had to be an integer constant, which includes the possibility of an expression formed from constant integer values. 

int n = 8;
char cookies[1];    // valid
char cakes[2 + 5]; // valid, size is a constant expression
char pies[2*sizeof(long double) + 1]; // valid
char my_array[n];     // invalid prior to C99, a VLA after C99

The name of a character array, like any array name, yields the address of the first element of the array.

Therefore, the following holds: 

char car[10] = "test";
car == &car[0] , 
*car == 'T', and 
*(car+1) == car[1] == 'a'

Indeed, you can use pointer notation to set up a string.

For example: 

const char * pt1 = "Something is pointing at me.";

This declaration is very nearly the same as this one: 

const char ar1[] = "Something is pointing at me.";

The declarations is saying that both pt1 and ar1 are addresses of strings. In both cases, the quoted string itself determines the amount of storage set aside for the string. Nonetheless, the forms are not identical.

Array Versus Pointer

Initializing the array copies a string from static storage to the array, whereas initializing the pointer merely copies the address of the string.

The following code illustrates these points. 

#define MSG "Hi'"

#include <stdio.h>
int main()
{
    char ar[] = MSG;
    const char *pt = MSG;
    printf("address of \"Hi'\": %p \n", "Hi'");
    printf("        address ar: %p\n", ar);
    printf("        address pt: %p\n", pt);
    printf("    address of MSG: %p\n", MSG);
    printf("address of \"Hi'\": %p \n", "Hi'");

    return 0;
}

Output:

pt and MSG are the same address, while ar is a different address.

Although the string literal "Hi'." occurs twice in the printf() statements, the compiler chose to use one storage location, but not the same address as MSG.

The compiler can store a literal that's used more than once in one or more locations.

The part of memory used for static data is different from that used for dynamic memory, the memory used for ar.

Array and Pointer Differences

For example, consider these two declarations: 

char my_string[] = "I love C!";
const char *my_string_pointer = "I love C++!";

The main difference is that the array name my_string is a constant, but the pointer my_string_pointer is a variable.

First, both can use array notation: 

for (i = 0; i < 6; i++)
    putchar(my_string[i]);
putchar('\n');
for (i = 0; i < 6; i++)
    putchar(my_string_pointer[i]));
putchar('\n');

This is the output: 

I love
I love

Next, both can use pointer addition: 

for (i = 0; i < 6; i++)
    putchar(*(my_string + i));
putchar('\n');
for (i = 0; i < 6; i++)
    putchar(*(my_string_pointer + i));
putchar('\n');

the output is as follows: 

I love
I love

Only the pointer version, however, can use the increment operator: 

while (*(my_string_pointer) != '\0')     /* stop at end of string            */
    putchar(*(my_string_pointer++));     /* print character, advance pointer */

This produces the following output: 

I love C++!

Suppose you want assign my_string_pointer with my_string. You can do the following: 

/* my_string_pointer now points to the array my_string */
my_string_pointer = my_string;

This makes the my_string_pointer pointer point to the first element of the my_string array.

However, you cannot say 

my_string = my_string_pointer;    /* illegal construction */

The situation is analogous to x = 3; versus 3 = x;.

The left side of the assignment statement must be a variable or, more generally, a modifiable lvalue, such as *p_int.

Incidentally, my_string_pointer = my_string; does not make the Millie string vanish; it just changes the address stored in my_string_pointer.

Unless you've saved the address of "I love C++!" elsewhere, however, you won't be able to access that string when my_string_pointer points to another location. There is a way to alter the my_string message-go to the individual array elements: 

my_string[7]= 'M';

or 

*(my_string + 7) = 'M';

The elements of an array are variables unless the array was declared as const, but the name is not a variable.
