C Choosing the Correct Data Type for the calculation

PreviousNext

You have to be careful to select the type of variable that you're using in your calculations so that it accommodates the range of values you expect.

If you use the wrong type, you will get errors.

Here's an example that shows how things can go wrong if you choose an unsuitable type for your variables: 

#include <stdio.h> 

int main(void) 
{ 
  const float delta = 4.5f; 
  short css = 43500;  
  short html = 19300; 
  short java = 21600; 
  float total = 0.0f; 

  short langTotal = css + html + java;     // Calculate total 

  printf("sold in\n CSS: %d\n HTML: %d\n Java: %d\n", css, html, java); 
  printf("Lang Total: %d\n", langTotal); 

  total = langTotal/150*delta; 
  printf("total is:$%.2f\n", total); 
  return 0; 
}

Output:

These are fairly simple calculations and you can see the Lang total is wrong.

langTotal variable is too small to hold the value and CSS value is too small as well.

A short variable can hold is 32767.

Solving the Problem

Try changing the program and running it again. 

#include <stdio.h> 

int main(void) 
{ 
  const float delta = 4.5f; 
  int css = 43500;  
  int html = 19300; 
  int java = 21600; 
  float total = 0.0f; 

  long langTotal = css + html + java;     // Calculate total 

  printf("sold in\n CSS: %d\n HTML: %d\n Java: %d\n", css, html, java); 
  printf("Lang Total: %ld\n", langTotal); 

  total = langTotal/150*delta; 
  printf("total is:$%.2f\n", total); 
  return 0; 
}
