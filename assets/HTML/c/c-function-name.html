C function name

PreviousNext

The name of a function can be any legal name in C that isn't a reserved word (such as int, double, sizeof, etc.).

And it cannot be the same as the name of another function in your program.

You should ensure that you do not use the same names as any of the standard library functions.

One way of differentiating your function names from those in the standard library is to start them with a capital letter.

Names starting with a capital letter are often reserved for use as struct type names.

A legal name has the same form as that of a variable:

  • a sequence of letters and digits,
  • the first of which must be a letter.

The underline character counts as a letter.

The name that you choose should give some clue as to what the function does and should not be too long.

Examples of valid function names that you might create are as follows:

  • cube_root
  • FindLast
  • findNext
  • ExplosionTest
  • Back2FrontPointer

Long name

You'll often want to define function names (and variable names too) that consist of more than one word.

There are three common approaches you can adopt that happen to be illustrated in the first three examples:

  • Separate each of the words in a function name with an underline character
  • Capitalize the first letter of each word
  • Capitalize words after the first

All three approaches work well, but the third form of name is more often used for variables.

You should pick an approach and stick to it.
