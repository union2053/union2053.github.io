C & Operator

PreviousNext

C the pointer is a variable used to store an address.

The unary & operator gives you the address where a variable is stored.

If my_int is the name of a variable, &my_int is the address of the variable.

You can think of the address as a location in memory.

Suppose you have the following statement: 

int my_int = 24;

Suppose that the address where my_int is stored is 0B76321.

Then the statement 

printf("%d %p\n", my_int, &my_int);

would produce this, %p is the specifier for addresses: 

24 0B76321

Example

The following code uses address & operator to see where variables of the same name- but in different functions-are kept. 

#include <stdio.h>
void myFunc(int);  /* declare function */
int main(void)
{
    int my_int = 2, bah = 5; /* local to main() */
    printf("In main(), my_int = %d and &my_int = %p\n",
              my_int, &my_int);
    printf("In main(), bah = %d and &bah = %p\n",
              bah, &bah);
    myFunc(my_int);

    return 0;
}

void myFunc(int bah) /* define function */
{
    int my_int = 10; /* local to myFunc() */

    printf("In myFunc(), my_int = %d and &my_int = %p\n",
              my_int, &my_int);
    printf("In myFunc(), bah = %d and &bah = %p\n",
              bah, &bah);
}

Output:

First, the two my_ints have different addresses.

The same is true for the two bahs.

So, as promised, the computer considers them to be four separate variables.

Second, the call myFunc(my_int) did convey the value (2) of the actual argument (my_int of main()) to the formal argument (bah of myFunc()).
