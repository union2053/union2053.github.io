C Defining a Structure Variable

PreviousNext

Consider the following code: 

struct book {/* structure template: tag is book*/
    char title[MAXTITL];
    char author[MAXAUTL];
    float value;
};

The structure definition tells the compiler how to represent the data, but it doesn't make the computer allocate space for the data.

The next step is to create a structure variable.

The following line causes a structure variable to be created: 

struct book library;

The compiler creates the variable library.

Using the book template, the compiler allots space for a char array of MAXTITL elements, for a char array of MAXAUTL elements, and for a float variable.

This storage is allocated together under the single name library.

In declaring a structure variable, struct book plays the same role that int or float does in simpler declarations.

For example, you could declare two variables of the struct book type or even a pointer to that kind of structure: 

struct book book1, book2, * bookPtr;

The structure variables book1 and book2 would each have the parts title, author, and value.

The pointer bookPtr could point to book1, book2, or any other book structure.

In essence, the book structure declaration creates a new type called struct book.

As far as the computer is concerned, the declaration struct book library; is short for 

struct book {
   char title[MAXTITL];
   char author[AXAUTL];
   float value;
} library;      /* follow declaration with variable name */

In other words, the process of declaring a structure and the process of defining a structure variable can be combined into one step.

Combining the declaration and the variable definitions, as shown here, is the one circumstance in which a tag need not be used: 

struct {         /* no tag */
    char title[MAXTITL];
    char author[MAXAUTL];
    float value;
} library;

Use the tag form, however, if you plan to use a structure template more than once, or you can use the typedef alternative.
