Question

Assume all variables are of type int.

Find the value of each of the following variables: 

a. x = (2 + 3) * 6;

b. x = (12 + 6)/2*3;

c. y = x = (2 + 3)/4;

d. y = 3 + 2*(x = 7/2);


a.      30 .
b.      27  (not  3 ).  (12 + 6)/(2*3)  would give  3 .
c.      x = 1, y = 1  (integer division).
d.      x = 3  (integer division) and  y = 9 .

Question 2

Assume all variables are of type int.

Find the value of each of the following variables: 

a. x = (int) 3.8 + 3.3;

b. x = (2 + 3) * 10.5;

c. x = 3 / 5 * 22.0;

d. x = 22.0 * 3 / 5;


a. 6  (reduces to  3 + 3.3 )
b. 52
c. 0  (reduces to  0 * 22.0 )
d. 13  (reduces to  66.0 / 5  or  13.2  and is then assigned to  int )

a. 30.0 / 4.0 * 5.0;

b. 30.0 / (4.0 * 5.0);

c. 30 / 4 * 5;

d. 30 * 5 / 4;

e. 30 / 4.0 * 5;

f. 30 / 4 * 5.0;

c. 35    (reduces to  7 * 5 )

e. 37.5  (reduces to  7.5 * 5 )
f. 35.0  (reduces to  7 * 5.0 )

Question 4

Find the error in the following code: 

int main(void)
{
  int i = 1,
  float n;
  printf("Watch out! Here come a bunch of fractions!\n");
  while (i < 30)
    n = 1/i;
    printf(" %f", n);
    printf("That's all, folks!\n");
    return;
}


Line 0 : Should include <stdio.h> .
Line 3 : Should end in a semicolon, not a comma.
Line 6 : The while  statement sets up an infinite 
         loop because the value of  i  remains  1
         and is always less than 30. Presumably, 
          we meant to write  while(i++ < 30) .
           
Lines 6-8: The indentation implies that we wanted lines 7 and 8 
           to form a block, but the
          lack of braces means that the while  loop includes only line 7. 
          Braces should be added.
           
Line 7 : Because  1  and  i  are both integers, the result of the 
         division will be  1  when  i  is  1,
         and 0  for all larger values. 
         Using  n = 1.0/i;  would cause  i  to be converted to floating
         point before division and would yield nonzero answers.

Line 8 : We omitted a newline character ( \n ) in the control statement. 
         This causes the numbers to be printed on one line, if possible.

Line 10 : Should be return 0; .
                      
Here is a corrected version:

#include <stdio.h>
int main(void)
{
  int i = 1;
  float n;
  printf("Watch out! Here come a bunch of fractions!\n");
  while (i++ < 30)
  {
    n = 1.0/i;
    printf(" %f\n", n);
  }
  printf("That's all, folks!\n");
  return 0;
}

Question 5

What will this program print? 

#include <stdio.h>
#define FORMAT "%s! C is cool!\n"
int main(void)
{
     int num = 10;

       printf(FORMAT,FORMAT);
       printf("%d\n", ++num);
       printf("%d\n", num++);
       printf("%d\n", num--);
       printf("%d\n", num);
       return 0;
}


%s! C is cool!
! C is cool!
11
11
12
11

The first  printf()  statement is the same as this:

printf("%s! C is cool!\n","%s! C is cool!\n");

The second print statement first increments  num  to  11  
and then prints the value. 

The third print statement prints num , which is  11, 
and then increments it to  12. 

The fourth print statement prints the current value of n, 
which still is  12, and then decrements  n  to 11. 

The final print statement prints the current value of  num , which is  11 .

Question 6

What will the following program print? 

#include <stdio.h>
int main(void)
{
     char c1, c2;
     int diff;
     float num;

         c1 = 'S';
         c2 = 'O';
         diff = c1 - c2;
         num = diff;
         printf("%c%c%c:%d %3.2f\n", c1, c2, c1, diff, num);
         return 0;
}


Here is the output:

SOS:4 4.00

The expression  c1 - c2  has the same value as  'S' - '0', 
which in ASCII is  83 - 79 .
