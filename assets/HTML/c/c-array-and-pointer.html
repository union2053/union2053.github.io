C Array and Pointer

PreviousNext

An array name by itself refers to an address: 

//Arrays and pointers 
#include <stdio.h> 

 int main(void) 
 { 

      char my_array[] = "My string"; 
    
      char *p = &my_array[0]; 
      printf("The address of the first array element   : %p\n", p); 
    
      p = my_array; 
      printf("The address obtained from the array name: %p\n", my_array); 
      return 0; 
}

Output:

Note

From the address we can see that the expression &my_array[0] produces the same value as the expression my_array.

Because my_array evaluates to the address of the first byte of the array and &my_array[0] evaluates to the first byte of the first element of the array.

They should be the same.

If you set p to my_array, which has the same value as &my_array[0], what does p + 1 correspond to?

Incrementing a pointer to an array of integers 

#include <stdio.h> 

int main(void) 
{ 
  long my_array[] = {15L, 25L, 35L, 45L}; 
  long *p = my_array; 

  for(int i = 0 ; i < sizeof(my_array)/sizeof(my_array[0]) ; ++i) 
    printf("address p+%d (&my_array[%d]): %llu        *(p+%d)   value: %d\n", 
                              i, i, (unsigned long long)(p+i),  i,  *(p+i)); 

  printf("\n    Type long occupies: %d bytes\n", (int)sizeof(long)); 
  return 0; 
}

Output:

Note

The pointer, p, is set to the address that results from my_array, where my_array is an array of elements of type long.

The pointer will initially contain the address of the first byte in the array, which is also the first byte of the element my_array[0].

This time the addresses are displayed using the %llu conversion specification after casting them to type unsigned long long so they will be decimal values.

The compiler presumes that when you add 1 to an address value, you actually want to do is access the next variable of that type.

On a computer that stores type long in 8 bytes, incrementing a pointer to long by 1 will increase the address value by 8.

Note that you could use the array name directly in this example.

You could write the for loop as 

for(int i = 0 ; i < sizeof(my_array)/sizeof(my_array[0]) ; ++i) 
    printf("address p+%d (&my_array[%d]): %llu        *(p+%d)   value: %d\n", 
             i, i, (unsigned long long)(my_array+i),  i,  *(my_array+i));

This works because the expressions my_array and my_array+i both evaluate to an address.

We output the values of these addresses and output the value at these addresses by using the * operator.

The arithmetic with addresses works the same here as it did with the pointer p.

Incrementing my_array by 1 results in the address of the next element in the array, which is 8 bytes further along in memory.

An array name refers to a fixed address and is not a pointer.

You can use an array name and thus the address it references in an expression, but you cannot modify it.
