C Precedence of Relational Operators

PreviousNext

The precedence of the relational operators is less than that of the arithmetic operators, including + and -, and greater than that of assignment operators.

This means, for example, that 

x > y + 2

means the same as 

x > (y + 2)

It also means that 

x = y > 2

means 

x = (y > 2)

In other words, x is assigned 1 if y is greater than 2 and is 0 otherwise; x is not assigned the value of y.

The relational operators have a greater precedence than the assignment operator.

Therefore, 

x_bigger = x > y;

means 

x_bigger = (x > y);

The relational operators are themselves organized into two different precedences. 

Higher precedence group:  < <= > >=
Lower precedence group:   == !=

The relational operators associate from left to right.

Therefore, 

ex != wye == zee

is the same as 

(ex != wye) == zee

First, C checks to see whether ex and wye are unequal.

Then, the resulting value of 1 or 0 (true or false) is compared to the value of zee.

The following table shows the priorities of the operators.

Operators (From High to Low Precedence)Associativity
() L-R
- + ++ -- sizeof R-L (type) (all unary)
* /% L-R
+ -L-R
< > <= >= L-R
== != L-R
= R-L
