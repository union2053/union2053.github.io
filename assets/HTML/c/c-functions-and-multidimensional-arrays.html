C Functions and Multidimensional Arrays

PreviousNext

The following code shows how to pass multi dimensional array to a function: 

#include <stdio.h>
#define ROWS 3//from w w w .dem o2s . c om
#define COLS 4
void sum_rows(int ar[][COLS], int rows);
void sum_cols(int [][COLS], int );    // ok to omit names
int sum2d(int (*ar)[COLS], int rows); // another syntax
int main(void)
{
     int my_array[ROWS][COLS] = {
            {2,4,6,8},
            {3,5,7,9},
            {12,10,8,6}
     };

     sum_rows(my_array, ROWS);
     sum_cols(my_array, ROWS);
     printf("Sum of all elements = %d\n", sum2d(my_array, ROWS));

     return 0;

}

void sum_rows(int ar[][COLS], int rows)
{
    int r;
    int c;
    int tot;

    for (r = 0; r < rows; r++)
    {
        tot = 0;
        for (c = 0; c < COLS; c++)
            tot += ar[r][c];
        printf("row %d: sum = %d\n", r, tot);
    }
}

void sum_cols(int ar[][COLS], int rows)
{
    int r;
    int c;
    int tot;

    for (c = 0; c < COLS; c++)
    {
        tot = 0;
        for (r = 0; r < rows; r++)
            tot += ar[r][c];
        printf("col %d: sum = %d\n", c, tot);
    }
}

int sum2d(int ar[][COLS], int rows)
{
    int r;
    int c;
    int tot = 0;

    for (r = 0; r < rows; r++)
        for (c = 0; c < COLS; c++)
            tot += ar[r][c];

    return tot;
}

Here is the output:

The program passes as arguments the name my_array, which is a pointer to the first element, a subarray, and the symbolic constant ROWS, representing 3, the number of rows.

Each function then treats ar as an array of arrays of four ints.

The number of columns is built in to the function, but the number of rows is left open.

The same function will work with, say, a 12 by 4 array if 12 is passed as the number of rows.

That's because rows is the number of elements; however, because each element is an array, or row, rows becomes the number of rows.

ar is used in the same fashion as my_array is used in main(). This is possible because ar and my_array are the same type: pointer-to-array-of-four-ints.

Be aware that the following declaration will not work properly: 

int sum2(int ar[][], int rows); // faulty declaration

For the compiler to evaluate this, it needs to know the size object to which ar points.

The declaration 

int sum2(int ar[][4], int rows); // valid declaration

says that ar points to an array of four ints, so ar+1 means "add 16 bytes to the address."

With the empty-bracket version, the compiler would not know what to do.

You can also include a size in the other bracket pair, as shown here, but the compiler ignores it: 

int sum2(int ar[3][4], int rows); // valid declaration, 3 ignored

In general, to declare a pointer corresponding to an N-dimensional array, you must supply values for all but the leftmost set of brackets: 

int sum4d(int ar[][12][20][30], int rows);

That's because the first set of brackets indicates a pointer, whereas the rest of the brackets describe the type of data object being pointed to, as the following equivalent prototype illustrates: 

int sum4d(int (*ar)[12][20][30], int rows);         // ar a pointer

Here, ar points to a 12 by 20 by 30 array of ints.
