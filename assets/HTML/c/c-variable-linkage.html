C Variable Linkage

PreviousNext

A C variable has one of the following linkages:

  1. external linkage,
  2. internal linkage, or
  3. no linkage.

No linkage

Variables with block scope, function scope, or function prototype scope have no linkage.

That means they are private to the block, function, or prototype in which they are defined.

external/internal linkage

A variable with file scope can have either internal or external linkage.

A variable with external linkage can be used anywhere in a multi-file program.

A variable with internal linkage can be used anywhere in a single translation unit.

static

To tell whether a file scope variable has internal or external linkage, check if the storage class specifier static is used in the external definition: 

int my_variable1 = 5;          // file scope, external linkage
static int my_variable2 = 3;  // file scope, internal linkage
int main()
{
    ...
}
...

The variable my_variable1 can be used by other files that are part of the same program.

The my_variable2 variable is private to this particular file, but can be used by any function in the file.
