C Unions

PreviousNext

A union is a type that enables you to store different data types in the same memory space not at the same time.

By using an array of unions, you can create an array of equal-sized units, each of which can hold a variety of data types.

Unions are set up in much the same way as structures.

There is a union template and a union variable.

They can be defined in one step or, by using a union tag, in two.

Here is an example of a union template with a tag: 

union my_Union {
    int digit;
    double my_variable;
    char letter;
};

This union above can hold an int value or a double value or a char value.

Here is an example of defining three union variables of the my_Union type: 

union my_Union fit;      // union variable of hold type
union my_Union save[10]; // array of 10 union variables
union my_Union * pu;     // pointer to a variable of hold type

The first declaration creates a single variable, fit.

The compiler allots enough space so that it can hold the largest of the described possibilities.

In this case, the biggest possibility listed is double, which requires 64 bits, or 8 bytes, on our system.

The second declaration creates an array called save with 10 elements, each 8 bytes in size.

The third declaration creates a pointer that can hold the address of a my_Union union.

Initialize a union

You have three choices:

  1. You can initialize a union to another union of the same type,
  2. You can initialize the first element of a union, or,
  3. with C99, you can use a designated initializer:
union my_Union valA;
valA.letter = 'R';
union my_Union valB = valA; // initialize one union to another
union my_Union valC = {88}; // initialize digit member of union
union my_Union valD = {.bigfl = 118.2}; // designated initializer

Using Unions Here is how you can use a union: 

fit.digit = 23;   // 23 is stored in fit; 2 bytes used
fit.my_variable = 2.0; // 23 cleared, 2.0 stored; 8 bytes used
fit.letter = 'h'; // 2.0 cleared, h stored; 1 byte used

The dot operator shows which data type is being used.

Only one value is stored at a time.

You can't store a char and an int at the same time, even though there is enough space to do so.

We have to write the program so that it keeps track of the data type currently being stored in a union.

You can use the -> operator with pointers to unions in the same fashion that you use the operator with pointers to structures: 

pu = &fit;
x = pu->digit;    // same as x = fit.digit

The next sequence shows what not to do: 

fit.letter = 'A';
double d = 3.02 * fit.my_variable;// ERROR

This code is wrong because a char type is stored, but the next line assumes that the content of fit is a double type.

However, sometimes it can be useful to use one member to place values into a union and to then use a different member for viewing the contents.

Anonymous Unions

Anonymous unions work much the same as anonymous structures.

An anonymous union is an unnamed member union of a structure or union.

For instance, we can define the structure using an anonymous union: 

struct owner {
     char id[12];
     ...
};

struct location {
     char name[40];
     char address[40];
     ...
};

struct car_data {
     char make[15];
     int status; /* 0 = owned, 1 = leased */
     union {
         struct owner owncar;
         struct location leasecar;
     };
...
};

Now, if my_car is a car_data structure, we can use *my_car.owncar.id* instead of
*my_car.ownerinfo.owncar.id*.
