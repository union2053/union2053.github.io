C External Variable Example

PreviousNext

Suppose you want two functions, call them main() and my_function(), to have access to the variable my_variable.

You can do this by declaring my_variable outside of and above the two functions. 

/* global.c -- uses an external variable */
#include <stdio.h>
int my_variable = 0;        /* an external variable      */
void my_function(void);
int main(void)
{
    extern int my_variable; /* an optional redeclaration */

    printf("Number please:\n");
    scanf("%d", &my_variable);
    while ( my_variable != 5)
        my_function();
    printf("hi!\n");

    return 0;
}

void my_function(void)
{
    /* optional redeclaration omitted */
    printf("Try again.\n");
    scanf("%d", &my_variable);
}

Output:

Note how the second value for my_variable was read by the my_function() function, yet main() also knew the new value when it finished the while loop.

So both the main() function and the my_function() function use the identifier my_variable to access the same variable.

In C terminology, my_variable has file scope, external linkage, and static storage duration.

We made my_variable an external variable by defining it outside of any function definition.

That's all you need to do to make my_variable available to all the subsequent functions in the file.

First, declaring my_variable where it is declared makes it available to the functions below it without any further action taken. Therefore, my_functions() uses the my_variable variable.

Similarly, nothing needed to be done to give main() access to my_variable. However, main() does have the following declaration in it: 

extern int my_variable;

In the example, this declaration is mainly a matter of documentation.

The storage class specifier extern tells the compiler that any mention of my_variable in this particular function refers to a variable defined outside the function, perhaps even outside the file.

Both main() and my_function() use the externally defined my_variable.
