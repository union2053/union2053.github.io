C Two-Dimensional Arrays And Pointers

PreviousNext

Let's look at some of the addresses related to the my_array array with this example:

Two-dimensional arrays and pointers 

#include <stdio.h> 

int main(void) 
{ 
  char my_array[3][3] = { 
                       {'1','2','3'}, 
                       {'4','5','6'}, 
                       {'7','8','9'} 
                     }; 

  printf("address of my_array         : %p\n", my_array); 
  printf("address of my_array[0][0]   : %p\n", &my_array[0][0]); 
  printf("value of my_array[0]        : %p\n", my_array[0]); 
  return 0; 
}

Output:

Note

All three output values are the same.

When you declare a one-dimensional array as x[n1], the [n1] after the array name tells the compiler that it's an array with n1 elements.

When you declare a two-dimensional array as y[n1][n2], the compiler conceptually creates a one-dimensional array of size n1, in which each element is an array of size n2.

When you declare a two-dimensional array, you're creating an array of one-dimensional sub arrays.

Ao when you access the two-dimensional array using the array name with a single index value, my_array[0], you're referencing the address of one of the sub arrays.

Using the two-dimensional array name by itself references the address of the beginning of the array of sub arrays, which is also the address of the beginning of the first subarray.

For the following expressions: 

my_array 
my_array[0] 
&my_array[0][0]

They all have the same value, but they aren't the same thing:

  • my_array is the address of a two-dimensional array of char elements,
  • my_array[0] is the address of a one-dimensional array of char elements that is a subarray of my_array,
  • &my_array[0][0] is the address of an array element of type char.

The expression my_array[1] results in the same address as the expression &my_array[1][0].

This is so because the latter expression is the first element of the second subarray, my_array[1].

When you use pointer notation to get to the values within a two-dimensional array, you can use the indirection operator. 

#include <stdio.h> 
int main(void) 
{ 
     char my_array[3][3] = { 
                          {'1','2','3'}, 
                          {'4','5','6'}, 
                          {'7','8','9'} 
                        }; 
    
     printf("value of my_array[0][0] : %c\n", my_array[0][0]); 
     printf("value of *my_array[0]    : %c\n", *my_array[0]); 
     printf("value of **my_array      : %c\n", **my_array); 
     return 0; 
}

The output from this program is as follows:

You need to apply two indirection operators to get it: **my_array.

If you use only one *, you will get the address of the first element of the array of arrays, which is the address referenced by my_array[0].

The relationship between the multidimensional array and its sub arrays is shown in the following. 

my_array ---->my_array[0]--->my_array[0][0]   my_array[0][1]   my_array[0][2] 
              my_array[1]--->my_array[1][0]   my_array[1][1]   my_array[1][2] 
              my_array[2]--->my_array[2][0]   my_array[2][1]   my_array[2][2]

As the figure shows, my_array refers to the address of the first element in the array of sub arrays, and my_array[0], my_array[1], and my_array[2] refer to the addresses of the first element in each of the corresponding sub arrays.
