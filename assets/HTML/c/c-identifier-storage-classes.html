C Identifier Storage Classes

PreviousNext

Memory

C provides several different storage classes for storing data in memory.

All of the program so far stores data in chunk of memory.

A chunk of memory can hold one or more values.

Or a chunk of memory might not yet actually have a stored value, but it will be of the right size to hold an appropriate value.

We use variable name to access the chunk memory. 

int entity = 3;

This declaration creates an identifier called entity.

An identifier of a variable is a name that can be used to designate the contents of a particular memory.

The identifier entity is used to reference value that's stored in computer's memory.

This declaration puts a value to be stored in the memory, value 3.

We can also use pointer to reference memory: 

int * pt = &entity;

Duration

We can describe an value in memory in terms of its duration, which is how long it stays in memory.

We can also describe an identifier by its scope and its linkage, which indicate which parts of a program can use it.

The different storage classes offer different combinations of scope, linkage, and storage duration.

You can have

  • identifiers that can be shared over several files of source code,
  • identifiers that can be used by any function in one particular file,
  • identifiers that can be used only within a particular function, and even
  • identifiers that can be used only within a subsection of a function.

You can have identifiers that exist for the duration of a program and identifiers that exist only while the function containing them is executing.

With concurrent programming, you can have objects that exist for the duration of a particular thread.

You also can store data in memory that is allocated and freed explicitly by means of function calls.
